# Spec: meeting-summary

변환된 텍스트를 LLM으로 요약하는 기능

## ADDED Requirements

### Requirement: 회의 내용 요약 생성

시스템은 전사본 텍스트를 LLM API에 전달하여 회의 요약을 생성해야 한다.
요약에는 핵심 내용, 결정 사항, 액션 아이템이 포함되어야 한다.

#### Scenario: 성공적인 요약 생성

- **WHEN** 전사본 텍스트가 제공될 때
- **THEN** 시스템은 LLM을 호출하여 구조화된 요약을 반환한다

#### Scenario: 요약 형식

- **WHEN** 요약이 생성될 때
- **THEN** 요약에는 "핵심 내용", "결정 사항", "액션 아이템" 섹션이 포함된다

### Requirement: 기존 Chat LLM 인프라 재사용

시스템은 기존 Chat 기능의 LLM API 설정과 키를 재사용해야 한다.

#### Scenario: API 키 재사용

- **WHEN** 요약을 요청할 때
- **THEN** 시스템은 Chat 기능에 설정된 LLM API 키를 사용한다

#### Scenario: API 키 미설정

- **WHEN** LLM API 키가 설정되지 않았을 때
- **THEN** 시스템은 "LLM API 키를 설정해주세요" 메시지와 함께 설정 화면으로 안내한다

### Requirement: 요약 생성 진행 상태 표시

시스템은 요약 생성 중 로딩 상태를 표시해야 한다.

#### Scenario: 요약 생성 중 로딩

- **WHEN** LLM API 호출이 진행 중일 때
- **THEN** 시스템은 "요약 생성 중..." 로딩 인디케이터를 표시한다

### Requirement: 요약 결과 저장 옵션

시스템은 요약 결과를 전사본과 함께 또는 별도로 저장할 수 있어야 한다.

#### Scenario: 요약만 저장

- **WHEN** 사용자가 "요약만 저장"을 선택할 때
- **THEN** 시스템은 요약 내용만 포함된 마크다운 노트를 생성한다

#### Scenario: 전사본과 함께 저장

- **WHEN** 사용자가 "전체 저장"을 선택할 때
- **THEN** 시스템은 요약과 전사본을 모두 포함한 마크다운 노트를 생성한다
