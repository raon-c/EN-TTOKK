# Spec: meeting-notes-ui

회의록 업로드, 변환, 요약을 위한 UI 패널

## ADDED Requirements

### Requirement: MeetingNotesPanel 컴포넌트

시스템은 기존 패널 UI 패턴과 일관된 MeetingNotesPanel을 제공해야 한다.
패널은 사이드바에서 접근 가능해야 한다.

#### Scenario: 패널 표시

- **WHEN** 사용자가 사이드바에서 회의록 아이콘을 클릭할 때
- **THEN** MeetingNotesPanel이 표시된다

### Requirement: 오디오 파일 업로드 UI

시스템은 오디오 파일을 업로드할 수 있는 드래그앤드롭 영역과 파일 선택 버튼을 제공해야 한다.

#### Scenario: 파일 선택 버튼 클릭

- **WHEN** 사용자가 "파일 선택" 버튼을 클릭할 때
- **THEN** 시스템은 파일 선택 다이얼로그를 열고 오디오 파일 필터를 적용한다

#### Scenario: 드래그앤드롭 업로드

- **WHEN** 사용자가 오디오 파일을 드롭존에 끌어다 놓을 때
- **THEN** 시스템은 파일을 업로드하고 처리를 시작한다

### Requirement: 변환 진행률 표시

시스템은 STT 변환 진행률을 프로그레스 바로 표시해야 한다.

#### Scenario: 변환 중 프로그레스 바

- **WHEN** STT 변환이 진행 중일 때
- **THEN** 시스템은 프로그레스 바와 퍼센트 수치를 표시한다

#### Scenario: 변환 취소 버튼

- **WHEN** 변환이 진행 중일 때
- **THEN** 시스템은 "취소" 버튼을 표시한다

### Requirement: 전사본 및 요약 미리보기

시스템은 변환 완료 후 전사본과 요약을 미리 볼 수 있어야 한다.

#### Scenario: 전사본 표시

- **WHEN** STT 변환이 완료될 때
- **THEN** 시스템은 전사본을 스크롤 가능한 영역에 표시한다

#### Scenario: 요약 표시

- **WHEN** 요약 생성이 완료될 때
- **THEN** 시스템은 요약을 전사본 위에 표시한다

### Requirement: 노트 저장 기능

시스템은 변환된 회의록을 vault에 마크다운 노트로 저장할 수 있어야 한다.

#### Scenario: 노트 제목 입력

- **WHEN** 사용자가 저장하려고 할 때
- **THEN** 시스템은 노트 제목을 입력받는 필드를 표시한다 (기본값: "회의록-{날짜}")

#### Scenario: 노트 저장 완료

- **WHEN** 사용자가 "저장" 버튼을 클릭할 때
- **THEN** 시스템은 마크다운 노트를 vault에 생성하고 성공 메시지를 표시한다

### Requirement: 실시간 녹음 UI

시스템은 앱 내에서 직접 오디오를 녹음할 수 있는 UI를 제공해야 한다.

#### Scenario: 녹음 시작

- **WHEN** 사용자가 "녹음" 버튼을 클릭할 때
- **THEN** 시스템은 마이크 권한을 요청하고 녹음을 시작한다
- **AND** 녹음 중 상태와 경과 시간을 표시한다

#### Scenario: 녹음 중지

- **WHEN** 사용자가 녹음 중 "중지" 버튼을 클릭할 때
- **THEN** 시스템은 녹음을 중지하고 녹음된 오디오를 처리 대기 상태로 전환한다

#### Scenario: 녹음 취소

- **WHEN** 사용자가 녹음 중 "취소" 버튼을 클릭할 때
- **THEN** 시스템은 녹음을 중지하고 녹음된 내용을 폐기한다

#### Scenario: 마이크 권한 거부

- **WHEN** 사용자가 마이크 권한을 거부할 때
- **THEN** 시스템은 권한이 필요하다는 안내 메시지를 표시한다

### Requirement: 모델 다운로드 UI

시스템은 Whisper 모델 다운로드를 위한 명시적인 UI를 제공해야 한다.
사용자가 선택하기 전에는 자동으로 다운로드를 시작하지 않는다.

#### Scenario: 모델 미설치 상태 표시

- **WHEN** 패널을 열었을 때 모델이 설치되지 않은 경우
- **THEN** 시스템은 "모델 다운로드 필요 (466MB)" 안내와 "다운로드" 버튼을 표시한다

#### Scenario: 모델 다운로드 중

- **WHEN** 사용자가 다운로드를 시작한 후
- **THEN** 시스템은 다운로드 프로그레스 바, 용량 정보, "취소" 버튼을 표시한다

#### Scenario: 모델 준비 완료

- **WHEN** 모델이 이미 다운로드되어 있을 때
- **THEN** 시스템은 "모델 준비 완료" 상태를 표시한다

#### Scenario: 파일 업로드는 모델과 무관하게 활성화

- **WHEN** 모델 설치 여부와 관계없이
- **THEN** 시스템은 파일 업로드 영역을 항상 활성화 상태로 표시한다

#### Scenario: 모델 미설치 시 변환 버튼 동작

- **WHEN** 모델이 없는 상태에서 파일 선택 후 "변환" 버튼을 클릭할 때
- **THEN** 시스템은 "모델 다운로드가 필요합니다" 메시지와 다운로드 버튼을 표시한다
